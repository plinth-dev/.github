# Plinth RFCs

Major design changes that alter a public API surface, the substrate's default shape, or the project roadmap go through an RFC. Smaller changes — bug fixes, narrow features, docs — skip this and go directly as PRs.

## How to file

1. Copy [`0000-template.md`](./0000-template.md) to `NNNN-short-title.md`. Pick the next available number.
2. Open a PR adding it.
3. Discussion happens in the PR. The author drives revisions.
4. Maintainers merge once consensus is reached. The merge is the acceptance.

Accepted RFCs become the source of truth until superseded. The implementation lands as separate PRs that reference the RFC.
